Explain the concept of Responsive Design. What does the term mean?

Responsive Design is something that should be kept in mind while creating webpages and websites. Implementing Responsive Design means that if anybody is looking at your webpage, the content and styling will not break depending on what their screen size is. This creates a good experience for anybody using the website since the website will adjust based on how the user is viewing it while also making it simple and easy to browse even when switching from a desktop to a mobile phone.

How does responsive design relate to web technologies?

Web technologies relates to responsive designs because the web technologies are constantly updating and new techniques emerge. Somebody browsing on an older outdated browser or computer shouldn’t be left out of experiencing responsive designs on the website. For example, the web technology for creating something on the browser google chrome might be different than how the same thing is accomplished on the safari browser. Therefore, the web developer needs to keep in mind how to provide a great experience to most of their user base without making too many sacrifices.

What did you learn about building web apps for different devices through your research?

I learned a lot about experimenting with the webpage size and by shrinking the window into a smaller size, I can see how the page will react to someone browsing on a smaller screen. I then identify what areas of my page can be played around with like a font size for some text on the screen that I can change to relative units like vh or vw which represent a percent of the viewport that the user is seeing the webpage through. I also, did a lot of playing around with the inspect element tool on the browser which can also be reached quickly by pressing “ctrl + shift + i” and I can see how the website is made as well as see how the website would look like on a phone or a tablet very quickly. Also, something as simple as creating a glowing text can be done on the browser but depending on what browser people are using you would have to write different code, for example:

-webkit-animation: glow 1s ease-in-out infinite alternate;
-moz-animation: glow 1s ease-in-out infinite alternate;
animation: glow 1s ease-in-out infinite alternate;
